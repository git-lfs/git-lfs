= git-lfs-track(1)

== NAME

git-lfs-track - View or add Git LFS paths to Git attributes

== SYNOPSIS

[source,role=synopsis,subs="verbatim,quotes"]
----
*git lfs track* [_options_] [_<pattern>_...] [--min-size <size>]
----

== DESCRIPTION

Start tracking the given patterns(s) through Git LFS. The argument is
written to .gitattributes. If no paths are provided, simply list the
currently-tracked paths.

The https://git-scm.com/docs/gitattributes[gitattributes documentation]
states that patterns use the
https://git-scm.com/docs/gitignore[gitignore pattern rules] to match
paths. This means that patterns which contain asterisk (`*`), question
mark (`?`), and the bracket characters (`[` and `]`) are treated
specially; to disable this behavior and treat them literally instead,
use `--filename` or escape the character with a backslash.

== OPTIONS

`-v`::
`--verbose`::
   If enabled, have `git lfs track` log files which it will touch. Disabled by
   default.
`-d`::
`--dry-run`::
   If enabled, have `git lfs track` log all actions it would normally take
   (adding entries to .gitattributes, touching files on disk, etc) without
   performing any mutative operations to the disk.
+
`git lfs track --dry-run [files]` also implicitly mocks the behavior of
passing the `--verbose`, and will log in greater detail what it is
doing.
+
Disabled by default.
`-j`::
`--json`::
  Writes the currently tracked patterns as JSON to STDOUT, if the command
  exits successfully. Intended for interoperation with external tools.
  Can not be combined with any `<pattern>` arguments. If `--no-excluded` is
  also provided, that option will have no effect.
`--filename`::
  Treat the arguments as literal filenames, not as patterns. Any special glob
  characters in the filename will be escaped when writing the `.gitattributes`
  file.
`-l`::
`--lockable`::
  Make the paths 'lockable', meaning they should be locked to edit them, and
  will be made read-only in the working copy when not locked.
`--not-lockable`::
  Remove the lockable flag from the paths so they are no longer read-only unless
  locked.
`--no-excluded`::
  Do not list patterns that are excluded in the output; only list patterns that
  are tracked.
`--no-modify-attrs`::
   Makes matched entries stat-dirty so that Git can re-index files you wish to
   convert to LFS. Does not modify any `.gitattributes` file(s).
`--min-size`::
   Automatically track files that are larger than the given size. Size values
   can be specified in human-readable format (e.g., "1MB", "500KB", "1GB").
   Uses both decimal (KB, MB, GB) and binary (KiB, MiB, GiB) units. This option
   requires at least one `<pattern>` argument. The threshold is written as the
   `autotracksize` custom attribute on the given pattern(s) in `.gitattributes`
   (e.g., `*.dat filter=lfs ... autotracksize=<bytes>`). Use
   `git lfs track "*" --min-size <size>` to apply it to all files. Existing
   files larger than the threshold are touched so they will be re-indexed by
   Git on the next `git add`.
   +
   Multiple `autotracksize` thresholds can be set for different file patterns
   by editing `.gitattributes` directly, e.g.:
   +
   [source,role="secondary"]
   ----
   *.dat filter=lfs autotracksize=1000000
   *.txt filter=lfs autotracksize=500000
   * filter=lfs autotracksize=10000000
   ----
   +
    The `.gitattributes` file must be committed to the repository for the
    autotrack setting to apply to all clones.
    +
    The following files are always excluded from LFS tracking at the filter
    level and can never become LFS pointers, even if explicitly tracked:
    `.gitattributes`, `.gitignore`, `.gitmodules`, `.mailmap`, `.gitkeep`.
    +
    Additional files are excluded from automatic tracking by default via the
    `lfs.autotrackexclude` config option:
    +
    [source,role="secondary"]
    ----
    .gitlab-ci.yml   *.md             *.txt
    *.cfg            *.ini            .github/*
    ----
    +
    When `lfs.autotrackexclude` is set, it replaces these defaults entirely.

== EXAMPLES

* List the patterns that Git LFS is currently tracking:
+
`git lfs track`
* Configure Git LFS to track GIF files:
+
`git lfs track "*.gif"`
* Configure Git LFS to track PSD files and make them read-only unless
locked:
+
`git lfs track --lockable "*.psd"`
* Configure Git LFS to track the file named `project [1].psd`:
+
`git lfs track --filename "project [1].psd"`
* Automatically track all files larger than 10 megabytes:
+
`git lfs track "*" --min-size 10MB`
+
* Automatically track `.dat` files larger than 1 megabyte:
+
`git lfs track "*.dat" --min-size 1MB`
+
Then commit the configuration:
+
[source,role="primary"]
----
git add .gitattributes
git commit -m "enable autotrack for files larger than 10MB"
----
* Set different thresholds for different file types by editing `.gitattributes`
   directly:
+
[source,role="primary"]
----
# Track .dat files larger than 1MB, .txt files larger than 500KB,
# and everything else larger than 10MB
*.dat filter=lfs autotracksize=1000000
*.txt filter=lfs autotracksize=500000
* filter=lfs autotracksize=10000000
----
